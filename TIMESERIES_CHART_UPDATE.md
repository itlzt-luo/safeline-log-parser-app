## 🎯 新增功能

### 📈 请求时间趋势折线图

在统计页面添加了一个**全宽的时间趋势折线图**，显示请求随时间变化的趋势。

---

## 📊 图表特性

### 1. 三条趋势线

图表同时显示三条折线，提供全面的时间维度分析：

| 趋势线 | 颜色 | 数据内容 | 用途 |
|--------|------|---------|------|
| **总请求** | 🔵 紫色 (#667eea) | 所有请求总数 | 查看整体流量趋势 |
| **成功请求** | 🟢 绿色 (#43e97b) | 2xx-3xx 状态码 | 查看正常请求趋势 |
| **错误请求** | 🔴 粉红 (#fa709a) | 4xx-5xx 状态码 | 快速发现异常时段 |

### 2. 智能时间粒度

图表会根据数据的时间跨度**自动选择最合适的时间粒度**：

| 时间跨度 | 时间粒度 | 示例 | 适用场景 |
|---------|---------|------|---------|
| **≤ 1天** | 按小时 | 0:00, 1:00, 2:00... | 今天的流量分布 |
| **2-7天** | 按天 | 3/1, 3/2, 3/3... | 本周流量趋势 |
| **8-30天** | 按天 | 3/1, 3/2, 3/3... | 本月流量趋势 |
| **>30天** | 按周 | 第1周, 第2周... | 长期流量趋势 |

**自动调整**：
- 数据量少时显示更细的粒度（小时级）
- 数据量多时合并为更大的时间段（周级）
- 确保图表始终清晰易读

### 3. 交互式 Tooltip

鼠标悬停在折线上时，显示详细信息：

```
┌─────────────────────┐
│ 14:00              │  ← 时间点
├─────────────────────┤
│ 总请求: 125        │  ← 总数（紫色）
│ 成功: 118          │  ← 成功数（绿色）
│ 错误: 7            │  ← 错误数（红色）
└─────────────────────┘
```

### 4. 视觉效果

- **折线样式**：平滑曲线（monotone）
- **线条粗细**：2px
- **数据点**：4px 圆点
- **激活点**：6px 圆点（鼠标悬停时）
- **网格线**：虚线网格，便于读数

## 💡 使用场景

### 场景 1：发现流量高峰

**操作**：查看时间趋势图

**结果**：
- 紫色线显示流量高峰时段
- 发现每天 10:00-12:00 是访问高峰
- 可以据此优化服务器资源

### 场景 2：定位错误爆发时间

**操作**：关注红色（错误）折线

**结果**：
- 发现 14:00 错误请求激增
- 可能是某个功能出现问题
- 结合其他图表定位具体原因

### 场景 3：分析成功率趋势

**操作**：对比绿色（成功）和紫色（总请求）

**结果**：
- 成功请求接近总请求 → 系统健康
- 成功请求远低于总请求 → 可能有问题
- 可以计算不同时段的成功率

### 场景 4：周期性分析

**操作**：查看多天数据的趋势

**结果**：
- 发现工作日流量高，周末流量低
- 夜间流量减少，符合预期
- 识别异常的流量模式

### 场景 5：筛选后的趋势分析

**操作**：
1. 设置筛选条件（如特定域名）
2. 查看时间趋势图

**结果**：
- 图表自动更新，只显示筛选后的数据
- 分析特定域名的流量趋势
- 对比不同域名的访问模式

## 🔧 技术实现

### 数据处理流程

```javascript
filteredLogs (筛选后的日志)
    ↓
按时间排序
    ↓
计算时间跨度
    ↓
选择时间粒度（小时/天/周）
    ↓
按时间粒度分组统计
    ↓
计算每组的总数、成功数、错误数
    ↓
转换为图表数据格式
    ↓
渲染折线图
```

### 核心代码

#### 1. 时间粒度选择

```javascript
const timeRangeMs = lastLog.timestamp - firstLog.timestamp;
const days = timeRangeMs / (1000 * 60 * 60 * 24);

if (days <= 1) {
  // 按小时分组
  groupBy = (date) => `${new Date(date).getHours()}:00`;
} else if (days <= 30) {
  // 按天分组
  groupBy = (date) => `${d.getMonth() + 1}/${d.getDate()}`;
} else {
  // 按周分组
  groupBy = (date) => `第${weekNum}周`;
}
```

#### 2. 数据统计

```javascript
const timeGroups = {};
sortedLogs.forEach(log => {
  const timeKey = groupBy(log.timestamp);
  if (!timeGroups[timeKey]) {
    timeGroups[timeKey] = {
      time: timeKey,
      total: 0,
      success: 0,
      error: 0
    };
  }
  timeGroups[timeKey].total++;
  if (log.status >= 200 && log.status < 400) {
    timeGroups[timeKey].success++;
  } else if (log.status >= 400) {
    timeGroups[timeKey].error++;
  }
});
```

#### 3. 折线图配置

```javascript
<LineChart data={timeSeriesData}>
  <CartesianGrid strokeDasharray="3 3" />
  <XAxis dataKey="time" angle={-45} textAnchor="end" height={80} />
  <YAxis />
  <Tooltip content={CustomTooltip} />
  <Legend />
  <Line type="monotone" dataKey="total" stroke="#667eea" />
  <Line type="monotone" dataKey="success" stroke="#43e97b" />
  <Line type="monotone" dataKey="error" stroke="#fa709a" />
</LineChart>
```

### 性能优化

```javascript
const timeSeriesData = useMemo(() => {
  // 计算时间序列数据
  // ...
}, [filteredLogs]);
```

**优化点**：
- 使用 `useMemo` 缓存计算结果
- 只在 `filteredLogs` 变化时重新计算
- 避免每次渲染都重新处理数据

## 🎓 数据解读指南

### 1. 观察总请求趋势（紫色线）

**上升趋势**：
- ✅ 业务增长
- ✅ 营销活动成功
- ⚠️ 可能需要扩容

**下降趋势**：
- ⚠️ 用户流失
- ⚠️ 服务问题
- ✅ 正常的周期性变化（如周末）

**波动剧烈**：
- ⚠️ 不稳定的流量
- ⚠️ 可能遭受攻击
- ✅ 正常的峰谷变化

### 2. 对比成功与总请求（绿线 vs 紫线）

**绿线接近紫线**：
- ✅ 系统健康
- ✅ 错误率低
- ✅ 用户体验好

**绿线远低于紫线**：
- ⚠️ 高错误率
- ⚠️ 系统可能有问题
- 🔍 需要深入调查

### 3. 关注错误请求（红色线）

**红线平稳接近0**：
- ✅ 系统稳定
- ✅ 无明显错误

**红线突然上升**：
- 🚨 发生故障
- 🚨 需要立即处理
- 🔍 查看具体时间点

**红线持续高位**：
- 🚨 严重问题
- 🚨 影响用户体验
- 🔧 需要紧急修复
