# 统计页筛选功能 - 快速参考

## 📊 功能概览

### 筛选器位置
```
统计页顶部 → 筛选条件面板 → 5个筛选选项 + 重置按钮
```

---

## 🔍 筛选选项

| 筛选器 | 图标 | 选项 | 用途 |
|--------|------|------|------|
| **开始日期** | 📅 | 日期选择器 | 设置时间范围起点 |
| **结束日期** | 📅 | 日期选择器 | 设置时间范围终点 |
| **状态码** | 📊 | 全部/2xx/3xx/4xx/5xx | 按响应状态筛选 |
| **请求方法** | 🔧 | 全部/GET/POST/PUT/... | 按HTTP方法筛选 |
| **域名** | 🌍 | 全部/domain1/domain2/... | 按访问域名筛选 |

---

## 💡 使用示例

### 示例 1：查看今天的错误
```
开始日期: 2026-03-04
结束日期: 2026-03-04
状态码: 4xx 客户端错误 或 5xx 服务器错误
```

### 示例 2：分析特定域名的POST请求
```
域名: fn.lztyun.cn
请求方法: POST
```

### 示例 3：本周成功请求统计
```
开始日期: 2026-02-26
结束日期: 2026-03-04
状态码: 2xx 成功
```

### 示例 4：组合分析
```
开始日期: 2026-03-01
结束日期: 2026-03-04
状态码: 所有状态
请求方法: GET
域名: fn.lztyun.cn
```
结果：特定域名在指定时间段内的所有GET请求

## 🔄 重置功能

**触发条件**：设置任意筛选条件后，「🔄 重置筛选」按钮出现

**点击效果**：
- 清除所有筛选条件
- 恢复显示全部日志数据
- 统计和图表恢复为原始状态

---

## 📌 筛选结果提示

**格式**：
```
📌 显示 50 条记录（共 1000 条）
```

**含义**：
- 50 = 筛选后的记录数
- 1000 = 总记录数

**显示条件**：仅在有活动筛选时显示

## ⚡ 性能优化

### 缓存机制
- ✅ 唯一方法列表缓存（useMemo）
- ✅ 唯一域名列表缓存（useMemo）
- ✅ 筛选结果缓存（useMemo）

### 好处
- 避免重复计算
- 快速响应筛选变化
- 支持大数据集（1000+ 条记录）

## 🔗 与日志详情页的关系

### 相同点
- 使用相同的筛选逻辑
- 方法标准化规则一致
- 日期处理方式相同

### 不同点
| 功能 | 统计页 | 日志详情页 |
|-----|--------|-----------|
| **数据展示** | 统计图表 | 详细表格 |
| **筛选位置** | 页面顶部独立面板 | 表格上方工具栏 |
| **额外功能** | 统计卡片 | 分页、排序、详情弹窗 |
| **搜索框** | 无 | 有（搜索路径/IP） |

## 🎓 最佳实践

### DO ✅
- 先设置大范围筛选（如时间段）
- 再细化具体条件（如状态码、方法）
- 使用组合筛选进行深入分析
- 及时重置筛选避免混淆

### DON'T ❌
- 不要设置过窄的时间范围（可能无数据）
- 不要同时应用过多筛选（可能结果为空）
- 不要忘记查看筛选结果数量提示

## 🐛 故障排除

### 筛选后无数据显示
**原因**：筛选条件过于严格
**解决**：
1. 检查筛选结果提示「显示 0 条记录」
2. 逐个放宽筛选条件
3. 或点击「重置筛选」重新开始

### 图表显示异常
**原因**：数据量过少
**解决**：放宽筛选条件，确保至少有几十条数据

### 日期筛选不生效
**原因**：日志时间戳格式问题
**解决**：确保日志文件包含有效的时间戳字段
