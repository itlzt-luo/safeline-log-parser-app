### 2. 增强的筛选条件 🔍

#### a) 全文搜索
**位置**: 顶部搜索框
**功能**: 搜索路径、IP、方法、域名
**测试**:
```
输入: "125.121.52.66"  → 应该只显示该 IP 的请求
输入: "/websocket"    → 应该只显示 WebSocket 相关请求
输入: "GET"           → 应该只显示 GET 请求
```

#### b) 状态码过滤
**位置**: 第一个下拉菜单
**选项**: 
- 📊 所有状态
- ✅ 2xx 成功
- 🔄 3xx 重定向
- ⚠️ 4xx 客户端错误
- ❌ 5xx 服务器错误

**测试**:
```
选择 "4xx 客户端错误" → 应该只显示 404 等 4xx 状态的请求
```

#### c) 请求方法过滤
**位置**: 第二个下拉菜单
**功能**: 动态显示日志中出现的所有 HTTP 方法
**测试**:
```
选择 "GET" → 应该只显示 GET 请求
选择 "POST" → 应该只显示 POST 请求（如果有）
```

#### d) 日期范围筛选
**位置**: 日期过滤器区域（淡紫色背景）
**功能**: 指定开始和结束日期
**测试**:
```
开始日期: 2026-03-04
结束日期: 2026-03-04
→ 应该只显示 2026-03-04 当天的请求
```

### 3. 多维度排序 📊

**位置**: 第三个下拉菜单

**排序选项**:
1. ⏬ 时间降序（默认）- 最新的在前
2. ⏫ 时间升序 - 最早的在前
3. 📦 大小降序 - 最大响应在前
4. 📦 大小升序 - 最小响应在前
5. 📈 状态升序 - 200 → 404 → 500
6. 📉 状态降序 - 500 → 404 → 200

**测试**:
```
选择 "大小降序" → 第一行应该是响应体最大的请求
选择 "状态降序" → 状态码从大到小排列
```
