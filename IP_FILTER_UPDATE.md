## 🔧 技术实现

### IP 列表提取

**自动提取唯一 IP**：
```javascript
const uniqueIPs = useMemo(() => {
  return [...new Set(logs.map(log => log.clientIp))].sort((a, b) => {
    // IP地址排序
    const aParts = a.split('.').map(Number);
    const bParts = b.split('.').map(Number);
    for (let i = 0; i < 4; i++) {
      if (aParts[i] !== bParts[i]) {
        return aParts[i] - bParts[i];
      }
    }
    return 0;
  });
}, [logs]);
```

**排序规则**：
- 按 IP 地址的数字大小排序
- 例如：1.1.1.1 → 192.168.1.1 → 192.168.1.2
- 便于快速找到相关 IP

## 💡 使用场景

### 场景 1：追踪可疑 IP

**问题**：发现某个 IP 产生大量错误请求

**操作**：
1. 在统计页查看「Top 10 客户端 IP」图表
2. 发现某个 IP 请求次数异常高
3. 在 IP 筛选器中选择该 IP
4. 查看该 IP 的时间趋势、请求路径、状态码分布

**结果**：
- 确定是否为恶意攻击
- 分析攻击模式
- 决定是否需要封禁

### 场景 2：用户行为分析

**问题**：需要分析特定用户的访问行为

**操作**：
1. 获取用户的 IP 地址
2. 在详情页选择该 IP
3. 查看该用户的所有访问记录
4. 分析访问路径、时间分布

**结果**：
- 了解用户行为路径
- 优化用户体验
- 发现使用问题

### 场景 3：故障定位

**问题**：某个 IP 报告无法访问

**操作**：
1. 在详情页选择该 IP
2. 查看该 IP 的请求状态码
3. 检查错误时间点
4. 查看具体的错误路径

**结果**：
- 快速定位问题
- 确认是用户端还是服务端问题
- 提供针对性解决方案

### 场景 4：内网监控

**问题**：监控内网特定服务器的访问

**操作**：
1. 选择内网 IP（如 192.168.x.x）
2. 查看该服务器的请求趋势
3. 分析访问域名和路径
4. 监控错误率

**结果**：
- 确保内网服务正常
- 发现异常访问
- 容量规划

### 场景 5：组合分析

**问题**：分析特定 IP 在特定时间段的特定域名访问

**操作**：
1. 设置时间范围
2. 选择域名
3. 选择 IP
4. 查看详细统计

**结果**：
- 精确的多维度分析
- 深入了解访问模式

## ⚡ 性能优化

### useMemo 缓存

```javascript
// 只在 logs 变化时重新计算
const uniqueIPs = useMemo(() => {
  // 提取并排序 IP
}, [logs]);

// 只在筛选条件变化时重新过滤
const filteredLogs = useMemo(() => {
  // 过滤逻辑
}, [logs, ..., ipFilter]);
```

**优势**：
- 避免每次渲染都计算
- 大数据集性能优秀
- 响应速度快
